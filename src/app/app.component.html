Angular + ESLint + Husky

Automated code quality & commit protection

What is this?

  • ESLint checks your code for style and errors.
  • Husky blocks commits if ESLint finds issues.
  • lint-staged only lints files you are committing.

How to Install & Setup

  1. Install dependencies:
    npm install --save-dev eslint typescript-eslint-parser typescript-eslint-eslint-plugin angular-eslint-eslint-plugin angular-eslint-eslint-plugin-template angular-eslint-template-parser husky lint-staged
  2. Initialize ESLint:
    npx eslint --init (or use the config in this repo)
  3. Initialize Husky:
    npx husky install
  4. Add pre-commit hook:
    npx husky add .husky/pre-commit "npx lint-staged"
  5. Add lint-staged config to package.json: 
    {
  "lint-staged": {
    "*.{ts,html}": ["eslint --fix"]
  }
}

How to Use

  1. Start your app: ng serve
  2. Make a code change that breaks a lint rule.
  3. Stage the file: git add <file>
  4. Try to commit: git commit -m "test lint"
  5. If there are lint errors, the commit will be blocked!
  6. Fix the errors, stage, and commit again.

How it Works

  • On every commit, Husky runs lint-staged.
  • lint-staged runs ESLint (with auto-fix) on staged files.
  • If any errors remain, the commit is blocked until you fix them.
  • This keeps your codebase clean and consistent!
Try it: break a lint rule and try to commit!
